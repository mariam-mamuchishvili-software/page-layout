help Questions

What is the difference between a page and a builder?

The page is the outermost full-viewport wrapper — it fills the screen and sets the background canvas. The builder (the container div) lives inside it and constrains content to a readable max-width, centred with margin-inline: auto. Think of page as the wall and builder as the framed picture hanging on it.

What is a partial and why does it matter?

A partial is a named region of the page — main, aside, header, or footer. Each partial owns a distinct area of the layout. The row splits the canvas into columns and each column holds one or more partials. Partials define where content lives, not what the content looks like.

How is a block different from a component?

A block is a named content group — posts, events, or tasks — that collects related components under one labelled roof. A component is the smallest self-contained unit: a single post card, an event card, or a task item. Blocks provide the heading and container; components provide the repeatable content inside.

Why use data-layout-structure instead of plain class names?

Class names describe style (card, row, active). The data-layout-structure attribute describes role in the architecture — a completely separate concern. This means you can restyle every component without touching the HTML, and tooling or a CSS inspector overlay can query structural intent rather than visual appearance.
